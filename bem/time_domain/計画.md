## φ の二次曲面フィットによる remesh 判定の設計

### 目的
幾何曲率 θ だけでは平坦領域（水底など）の φ 急変を拾えない。
φ(u,v) の Hessian 固有値（φ の「主曲率」）を θ と同じ枠組みで計算し、
split/collapse 判定に使う。

### 計画ファイル
`/Users/tomoaki/.claude/plans/mutable-dazzling-wombat.md`
第1段階: 計算 + VTU 可視化のみ。remesh 判定への統合は第2段階。

### 設計方針（確定）
- `fitQuadricLocal` に4引数オーバーロード（高さ値を外から渡す）
- `hessianEigenvaluesFromQuadric`（slope correction なしの別関数）
- φ の CORNER 点はスキップ、近傍は同一 BC タイプに制限
- キャッシュなし（毎回全再計算）— φ は多箇所で直接更新されるため invalidation 困難
- VTU 出力直前に computePhiCurvatures() を呼ぶ（remesh 内だけだと stale になる）

### 未解決: CORNER 越しフィットの二律背反（2026-04-07 中断地点）

**幾何側の現状:**
- `computePrincipalCurvatures` は CORNER フィルタなしで全近傍を使用（意図的）
- crease 方向（PD2）を得るには両面の情報が必要 → フィルタできない
- しかし k1（二面角由来）が `edgeThetaVerdict` 経由で split 判定に漏れる

**`edgeCurvatureAngle` の方向分解:**
θ = sqrt(k1²(e·PD1)² + k2²(e·PD2)²)
- CORNER 辺（crease 沿い）: e·PD1 ≈ 0 → k1 が消える → 問題なし
- CORNER 点に接続する非 CORNER 辺（斜め）: e·PD1 ≠ 0 → k1 が漏れる → θ 膨張

**検討中の対策案:**
- CORNER 点の θ 計算で k1 項を除外（k2·(e·PD2) のみ）
- 端点が CORNER の場合は非 CORNER 側の端点の θ だけを使う
- φ 側は crease 方向不要なので CORNER スキップで問題なし

**Why:** CORNER 越しの曲率の大きさは feature（二面角/BC遷移）であり、
離散化誤差ではない。メッシュを細かくしても解消しない。

**How to apply:** φ 曲率実装前に、幾何側の k1 漏れ問題も含めて設計を固める。
